What is a Set?

Author: Billy Price

What is a set? A computer scientist might tell you its a kind of collection data-type, perhaps a list, with two special features:

  • The order of elements doesn’t matter.
  • Each element appears at most once in any set - OR - adding the same element multiple times doesn’t change what the set is.

Perhaps the data type should also be expected to check if something is in the set with reasonable efficiency. This leads to an expectation of work required to maintain the set when adding, removing and inspecting elements of the set.

You might wonder how this fits into the logic treatment of sets.

  • Must the elements of the set be orderable in order to have a set?
  • Does this re-ordering and duplicate-element-removal happen magically?
  • Are we allowed to have infinite sets? Why?

What a set actually is

The view we will take is that the only thing important about a set is the elementhood test, that is, a set, AAA, is whatever you want, as long as, given any object, xxx, we can answer true or false to the statement x∈Ax \in Ax∈A (xxx is an element of AAA). In fact, its best to just think of the data of a set as the elementhood test itself, i.e. a one-place-predicate, in the sense that there is nothing more that defines it. In the opposite direction, we can think of any one-place-predicate, P(x)P(x)P(x), as a set, but to dress it up we use this set-comprehension syntax,

{x∣P(x)}\{x \mid P(x) \}{x∣P(x)}

i.e., “the set of all xxx such that P(x)P(x)P(x)”. Despite this description, there is no actual obligation to gather all of these elements into some collection. We just have a thing which becomes the predicate that it holds when called on by the ∈\in∈ symbol. So when we posit a statement like

y∈{x∣P(x)}y \in \{ x \mid P(x)\}y∈{x∣P(x)}

we’re actually just saying P(y)P(y)P(y).

Given an anonymous set, AAA, you might imagine AAA to secretly have the form {x∣P(x)}\{x \mid P(x)\}{x∣P(x)} for some predicate P(x)P(x)P(x). Of course P(x)P(x)P(x) must always agree with x∈Ax \in Ax∈A, so we can bring this full circle by saying:

A={x∣x∈A}A = \{x \mid x \in A\}A={x∣x∈A}

Set equality and subset

This view on sets as predicates motivates what it means for two sets to be equal - it just means that their associated one-place-predicates (unary predicates) are the same. So given two predicates P(x)P(x)P(x) and Q(x)Q(x)Q(x), what do we mean when we say they are the same? How about, they have the same truth value for every possible xxx? We can package this as the assertion:

∀x (P(x)⇔Q(x))\forall x~(P(x) \Leftrightarrow Q(x))∀x (P(x)⇔Q(x))

which may or may not be true at some interpretation, I\mathcal{I}I.

By extension, given sets AAA and BBB, we define A=BA = BA=B to mean ∀x (x∈A⇔x∈B)\forall x~ (x \in A \Leftrightarrow x \in B)∀x (x∈A⇔x∈B) You can see that this reduces to the original statement when AAA is {x∣P(x)}\{ x \mid P(x)\}{x∣P(x)} and BBB is {y∣Q(y)}\{y \mid Q(y)\}{y∣Q(y)}.

Now what would it mean if we weakened the use of biimplication, ⇔\Leftrightarrow⇔, to implication, ⇒\Rightarrow⇒?

∀x (x∈A⇒x∈B)\forall x~(x \in A \Rightarrow x \in B)∀x (x∈A⇒x∈B)

This is precisely what we mean when we say A⊆BA \subseteq BA⊆B. Notice that we can combine A⊆BA \subseteq BA⊆B and B⊆AB \subseteq AB⊆A to conclude A=BA = BA=B, since

A⊆B∧B⊆A≡∀x (x∈A⇒x∈B)∧∀y (y∈B⇒y∈A)≡∀x [(x∈A⇒x∈B)∧(x∈B⇒x∈A)]≡∀x (x∈A⇔x∈B)≡A=B \begin{aligned} A \subseteq B \wedge B \subseteq A &\equiv \forall x~(x \in A \Rightarrow x \in B) \wedge \forall y~(y \in B \Rightarrow y \in A) \\ &\equiv \forall x~[(x \in A \Rightarrow x \in B) \wedge (x \in B \Rightarrow x \in A)] \\ &\equiv \forall x~(x \in A \Leftrightarrow x \in B) \\ &\equiv A = B \end{aligned}A⊆B∧B⊆A​≡∀x (x∈A⇒x∈B)∧∀y (y∈B⇒y∈A)≡∀x [(x∈A⇒x∈B)∧(x∈B⇒x∈A)]≡∀x (x∈A⇔x∈B)≡A=B​

Notice that A⊆BA \subseteq BA⊆B doesn’t rule out the possibility that A=BA = BA=B, and to express “proper subset”, A⊂BA \subset BA⊂B, we must say something like (A⊆B)∧∃x (x∈A∧x∉B)(A \subseteq B) \wedge \exists x~( x \in A \wedge x \not\in B)(A⊆B)∧∃x (x∈A∧x​∈B) or A⊆B∧A≠BA \subseteq B \wedge A \neq BA⊆B∧A​=B. Don’t be misled by the pronounciation “subset or equal to” for ⊆\subseteq⊆ sounding like a compound statement involving ⊂\subset⊂ “subset”, as a component - it’s really the other way around.

A few kinds of sets and constructions

Empty set

Perhaps the most important, yet often forgetten set, is the empty set. This is just the “always false” predicate, so we define it like this.

∅:={x∣f}\emptyset := \{x \mid \mathbf{f}\}∅:={x∣f}

HOT TIP: Whenever a question mentions some arbitrary-unspecified set, consider what would happen if it was the empty set.

Singleton sets

Given an object aaa, we can define the set just containing aaa, written {a}\{a\}{a}, like this:

{x∣x=a}\{x \mid x = a\}{x∣x=a}

Finite sets

Given some objects, say a,b,c,da,b,c,da,b,c,d, we can describe a set which contains those elements, and nothing else. {a,b,c,d}\{a,b,c,d\}{a,b,c,d} But is this the same as {a,a,d,c,b}\{a,a,d,c,b\}{a,a,d,c,b}? How would we demonstrate this? The problem is we haven’t defined what {a,b,c,d}\{a,b,c,d\}{a,b,c,d} is - what is the underlying predicate? Here’s a simple way to define it:

{a,b,c,d}:={x∣x=a∨x=b∨x=c∨x=d}\{a,b,c,d\} := \{x \mid x = a \vee x = b \vee x = c \vee x = d\}{a,b,c,d}:={x∣x=a∨x=b∨x=c∨x=d}

Now it’s clear that we could demonstrate {a,b,c,d}={a,a,d,c,b}\{a,b,c,d\} = \{a,a,d,c,b\}{a,b,c,d}={a,a,d,c,b}, since this is just proving that

(x=a∨x=b∨x=c∨x=d)≡(x=a∨x=a∨x=d∨x=c∨x=b)(x = a \vee x = b \vee x = c \vee x = d) \equiv (x = a \vee x = a \vee x = d \vee x = c \vee x = b)(x=a∨x=b∨x=c∨x=d)≡(x=a∨x=a∨x=d∨x=c∨x=b)

Infinite Sets

Gathering a complete collection containing infinitely many things is a questionable thing to do, but we do not run into this problem with sets. Why? Because we do not need to generate/nor collect any of the elements in the set, we just need to be able to answer yes or no whenever prompted with an object. If there are infinitely objects that we would say yes to, then we have an infinite set.

You might protest at my presumption that there are infinitely many things to test for elementhood, and you’d be absolutely right. Every foundational theory which rigorously defines what a set is, will have an axiom stating that there is at least one infinite set, usually the set of natural numbers N\mathbb{N}N.

Union and Intersection sets

The union of two sets is quite plainly related to the disjunction connective. Given sets AAA and BBB, we define

A∪B:={x∣x∈A∨x∈B}A \cup B := \{x \mid x \in A \vee x \in B\}A∪B:={x∣x∈A∨x∈B}

Looking back at the finite set example, notice that

{a,b,c,d}={a}∪{b}∪{c}∪{d}\{a,b,c,d\} = \{a\} \cup \{b\} \cup \{c\} \cup \{d\}{a,b,c,d}={a}∪{b}∪{c}∪{d}

Likewise, the intersection is constructed using the conjunction connective. Given sets AAA and BBB, we define

A∩B:={x∣x∈A∧x∈B}A \cap B := \{x \mid x \in A \wedge x \in B\}A∩B:={x∣x∈A∧x∈B}

Just like disjunction of logical formulas creates a formula where (usually) more models make it true, the union of sets create (usually) larger sets. Likewise, the intersection of sets create (usually) smaller sets, since the conjunction of logical formulas creates one which is true at (usually) less models.

The important difference between sets and unary predicates

Hopefully by now you are convinced that a set “is just a predicate”. However I want to make a careful distinction now. A set is a set, and only becomes a logical statement via the ∈\in∈ symbol. Likewise, a unary predicate, or any logical statement φ(x)\varphi(x)φ(x), is not a set, and only becomes a set once we lift it to a set by writing {x∣φ(x)}\{x \mid \varphi(x)\}{x∣φ(x)}.

Essentially, a set is a wrapper for a predicate, and that predicate is accessed via the ∈\in∈ symbol.

As an example, if AAA and BBB are sets, then the following statement does not make sense,

(A∩B)⇔(x∈B∧x∈A)\color{red}{(A \cap B)} \Leftrightarrow (x \in B \wedge x \in A)(A∩B)⇔(x∈B∧x∈A)

because the thing on the left side of the ⇒\Rightarrow⇒ is not a logical assertion. We cannot assert a set, we must assert something about a set.

Likewise the following statement doesn’t make sense either

(A∩B)=(x∈B∧x∈A)(A \cap B) = \color{red}{(x \in B \wedge x \in A)}(A∩B)=(x∈B∧x∈A)

since the thing on the right side of the equals sign should be a set, not a logical assertion. We can fix both sentences as follows.

(x∈A∩B)⇔(x∈B∧x∈A)(A∩B)={x∣x∈B∧x∈A}\begin{aligned}(x \in A \cap B) &\Leftrightarrow (x \in B \wedge x \in A) \\ (A \cap B) &= \{x \mid x \in B \wedge x \in A\} \end{aligned}(x∈A∩B)(A∩B)​⇔(x∈B∧x∈A)={x∣x∈B∧x∈A}​

For similar reasons, the reading of x∈A∩Bx \in A \cap Bx∈A∩B should be unambiguously x∈(A∩B)x \in (A \cap B)x∈(A∩B), and not (x∈A)∩B(x \in A) \cap B(x∈A)∩B, because we can only intersect sets, not logical statements like x∈Ax \in Ax∈A. Of course we can lift any logical statement to a set, but that would be written {x∣x∈A}∩B\{x \mid x \in A\} \cap B{x∣x∈A}∩B, which is just A∩BA \cap BA∩B, and likely not what we mean when we write x∈A∩Bx \in A \cap Bx∈A∩B.
